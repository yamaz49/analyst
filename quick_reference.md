# Universal Data Analyst - 速查卡

## 快速开始（三步使用）

```bash
# 1. 运行分析流程
python orchestrator.py data.csv --intent "你的分析诉求"

# 2. 获取生成的提示词文件
#    - step2_ontology_prompt.txt
#    - step4_planning_prompt.txt
#    - step5_script_prompt.txt

# 3. 将提示词发送给大模型，获取分析结果
```

## 数据本体识别速查

### 实体类型判断

| 特征 | 类型 | 示例 |
|------|------|------|
| 有时间戳、单次发生、不可重复 | **交易/事件型** | 订单、点击、地震 |
| 时间点快照、可累积 | **状态/存量型** | 库存、人口、余额 |
| 描述实体间连接 | **关系/网络型** | 社交关系、贸易流 |
| 静态属性、少随时间变 | **特征/属性型** | 用户画像、商品参数 |
| 连续测量、有序列依赖 | **时序/轨迹型** | 股价、气温、传感器 |

### 生成机制判断

| 机制 | 特点 | 潜在偏差 |
|------|------|---------|
| **观测型** | 被动记录 | 选择偏差、幸存者偏差 |
| **实验型** | 有干预/对照 | 较少偏差，可因果推断 |
| **模拟型** | 基于规则生成 | 结论仅限模拟场景 |
| **测量型** | 仪器采集 | 测量误差 |
| **报告型** | 人工填报 | 社会期望偏差 |

## 问题类型判定速查

| 类型 | 关键词 | 数据要求 | 典型方法 |
|------|--------|---------|---------|
| **描述型** | 是什么、分布、特征 | 代表性样本 | 描述统计、可视化 |
| **诊断型** | 为什么、原因、归因 | 多维度分解 | 分解分析、根因分析 |
| **预测型** | 会怎样、趋势、预警 | 时间序列 | ARIMA、Prophet、ML |
| **规范型** | 应怎样、最优、策略 | 行动-结果映射 | 优化模型、决策分析 |
| **因果型** | 有效应吗、影响、机制 | 对照组/时间变化 | DID、RDD、PSM |

## 经济类型速查

| 数据特征 | 经济类型 | 分析方法 |
|---------|---------|---------|
| 订单+价格+利润+SKU | **零售经济** | 价值链 + ABC-XYZ + RFM |
| 订单+price+B2B客户 | **混合零售** | 价值链 + 客户分层 + 价格弹性 |
| view+cart+buy事件链 | **注意力经济** | 漏斗分析 + 会话挖掘 |
| GMV+平台撮合 | **佣金经济** | 双边网络效应 + 单位经济 |
| 用户+订阅周期+Churn | **订阅经济** | LTV/Cohort + 留存曲线 |
| 资产+使用记录 | **租赁经济** | 资产利用率 + 收益管理 |
| 职位+技能+薪资 | **劳动力市场** | 技能溢价 + 经验弹性 |
| OHLCV价格 | **金融时序** | 技术分析 + 波动率建模 |

## 常用Python模式

### 加载数据
```python
from orchestrator import DataAnalysisOrchestrator

orch = DataAnalysisOrchestrator()
orch.step1_load_data("data.csv")
```

### 生成本体识别提示词
```python
prompt, file_path = orch.step2_identify_ontology()
# 将 prompt 发送给大模型
```

### 生成方案规划提示词
```python
prompt, file_path = orch.step4_plan_analysis("分析销售趋势")
# 将 prompt 发送给大模型
```

### 完整单轮分析
```python
results = orch.run_full_analysis(
    file_path="data.csv",
    user_intent="探索性数据分析",
    run_validation=True
)
```

## 常见问题

**Q: 为什么生成的是提示词而不是直接结果？**
A: 每次分析都需要大模型根据具体数据特征进行思考判断，不使用硬编码规则。

**Q: 如何使用生成的提示词？**
A: 将 .txt 文件内容复制给 Claude/ChatGPT/GPT-4 等大模型，获取结构化结果。

**Q: 支持哪些数据格式？**
A: CSV, Excel, Parquet, JSON, SQL数据库（通过data_loader.py）

**Q: 数据量有限制吗？**
A: 受内存限制，通常支持百万行级别。超大数据建议采样分析。

**Q: 可以分析非经济数据吗？**
A: 可以。技能会根据数据特征自动识别领域类型，匹配相应分析方法。

## 核心原则

> **先定本体，再选方法，最后执行。**
>
> **不问"这是什么经济"，问"这是关于什么的存在"。**
>
> **不问"怎么赚钱"，问"要解决什么认知问题"。**
